# CaptureLab 0.4.0 Development Plan

Date: 2026-07-03

## Goal

CaptureLab 0.4.0 should move the app from a narrow MVP into a reliable everyday macOS screenshot tool.

The release should keep the native, lightweight product direction. It should fix the real defects found in review, remove misleading UI, and add the missing capture-entry features that users expect from mature screenshot tools such as ShareX, Flameshot, ksnip, eSearch, macshot, and Snow Shot.

## Release Theme

Reliable capture, honest editor, faster repeat use.

## Scope Summary

### P0: Release Blockers

These must ship in 0.4.0.

1. Verify and harden mosaic Y-axis sampling.
   - Centralize pixelation for `NSImage+CaptureLab.captureLabPixelatedRegion` and `CaptureAnnotationCanvasView.pixelatedImage`.
   - Keep CGImage cropping in top-origin pixel coordinates: `y = normalizedRect.minY * height`.
   - Do not reuse the AppKit drawing formula `y = (1 - normalizedRect.maxY) * height` for CGImage cropping; live verification showed that would sample the vertically mirrored region.
   - Add a regression test with a known top/bottom color image so the sampled mosaic region proves it comes from the selected area.

2. Remove or implement the fake zoom menu.
   - Preferred 0.4.0 path: implement 50%, 100%, 200%, and fit-to-window zoom.
   - If implementation is not ready, remove `ZoomToolbarMenu` from the toolbar before release.
   - Acceptance: every visible zoom action changes the canvas scale or no zoom UI is shown.

3. Remove dead inspector code.
   - Delete the unused `CaptureInspectorView` surface from `OCRPanelView.swift`, or move any still-useful pieces behind a real feature entry.
   - Rename files if needed so file names match their remaining contents.
   - Acceptance: no unreferenced inspector UI remains in production sources.

4. Stabilize launch/window behavior.
   - Replace the current delayed title-string based main-window close logic with a deterministic window presentation flow.
   - Acceptance: launching CaptureLab opens only the menu bar item unless the user explicitly opens the editor.

### P1: Core Screenshot Tool Gaps

These define the main 0.4.0 user-facing upgrade.

1. Add a real global screenshot hotkey.
   - The configured shortcut must work while CaptureLab is in the background.
   - Keep the existing shortcut recorder UI, but make its wording explicit: global screenshot shortcut.
   - Validate collisions and show a clear error when registration fails.
   - Acceptance: from another active app, pressing the configured shortcut starts region capture.

2. Add basic capture modes.
   - Region capture remains the default.
   - Add full screen capture.
   - Add active/window capture if reliable through `screencapture` flags.
   - Add delayed region capture, at least 3s and 5s.
   - Add capture previous region if the prior region can be persisted safely.
   - Acceptance: menu bar and app menu expose the supported modes; every mode either captures correctly or fails with a clear localized error.

3. Add recent captures history.
   - Persist rendered PNGs and lightweight metadata under `~/Library/Application Support/CaptureLab/History`.
   - Keep a bounded history, for example last 30 captures or a configurable disk limit.
   - Menu bar should expose recent captures with open, copy, save as, and upload actions.
   - Acceptance: a capture remains available after app restart and can be copied without retaking the screenshot.

4. Add pin-to-screen as the history follow-up if time permits.
   - Implement floating always-on-top image windows for current capture and recent captures.
   - Keep this as a stretch item if history or global hotkey expands.
   - Acceptance: a pinned image stays above normal windows and can be closed with Esc or Command-W.

### P2: Editor Trust And Polish

These are smaller improvements that make the existing editor feel less MVP.

1. Centralize annotation commit thresholds.
   - Move the "large enough to commit" rule onto `CaptureAnnotation` or a shared helper.
   - Use one rule from both ViewModel and canvas paths.
   - Acceptance: tests cover text and non-text thresholds through the shared logic.

2. Fix counter numbering.
   - Avoid duplicate counter values after deleting a middle marker.
   - Prefer a monotonic per-document counter seed.
   - Acceptance: creating 1, 2, 3, deleting 2, and creating another counter produces 4.

3. Add minimal style controls.
   - At minimum: annotation color and line width for arrow, line, rectangle, and brush.
   - Defer full style presets, stickers, shadows, and rich text.
   - Acceptance: exported PNG matches the preview style.

4. Normalize capture async code.
   - Replace the GCD-based `captureRegion` path with `Task` plus a detached blocking capture call.
   - Avoid long-lived strong self captures where they are not necessary.
   - Acceptance: behavior is unchanged, code style matches upload/update async paths, and tests still pass.

5. Reduce redundant R2 normalization.
   - Normalize settings once in the upload path and pass the normalized settings through signing, object key generation, and public URL generation.
   - Acceptance: existing R2 signing and upload tests still pass.

## Explicit Non-Goals For 0.4.0

- Screen recording and GIF recording.
- Scrolling screenshot.
- OCR translation, search, or AI workflows.
- Multi-cloud upload destinations beyond the current Cloudflare R2 path.
- PDF or batch OCR.
- Full editor redesign.
- App Store sandboxing or notarization work, unless needed for distribution separately.

These are valid roadmap items, but they should not block 0.4.0.

## Suggested Implementation Order

1. Fix mosaic coordinate bugs and add regression tests.
2. Remove dead inspector code and resolve fake zoom UI by implementing or deleting it.
3. Stabilize app launch/window presentation.
4. Implement global hotkey registration and failure handling.
5. Add capture mode model and wire menu bar/app commands.
6. Add history storage and recent-capture actions.
7. Add optional pin-to-screen if the first six items are stable.
8. Polish annotation thresholds, counter numbering, async capture, and R2 normalization.
9. Run release validation and package both arm64 and x86_64 artifacts.

## Test Plan

- Unit tests:
  - Mosaic top/bottom sampling regression.
  - Shared annotation commit threshold.
  - Counter monotonic numbering.
  - History retention and metadata load/save.
  - Capture mode command construction where shell-out is still used.
  - R2 upload tests after normalization refactor.

- Manual QA:
  - Launch app: no editor window opens automatically.
  - Global hotkey works from Finder, Safari/Chrome, Terminal, and another full-screen app if possible.
  - Region, full screen, window, delayed capture, and previous-region capture work or fail clearly.
  - Copy, save, upload, and Done still render annotations correctly.
  - Mosaic preview and exported PNG obscure the selected content, not the vertically mirrored source region.
  - Recent capture survives restart.
  - Pinned capture stays above normal windows and closes cleanly if pin ships.

- Release validation:
  - `swift test` under full Xcode, not Command Line Tools only.
  - `./script/build_and_run.sh`.
  - `CAPTURELAB_ARCH=arm64 ./script/package_dmg.sh`.
  - `CAPTURELAB_ARCH=x86_64 ./script/package_dmg.sh`.
  - Mount each DMG and verify the app launches, menu bar item appears, global hotkey can be configured, and basic capture/edit/copy works.
  - Verify checksums before publishing.

## Version Definition Of Done

0.4.0 is ready when:

- No known fake or dead UI ships.
- The mosaic bug is fixed with a regression test.
- The app can be invoked globally without focusing CaptureLab first.
- At least region, full screen, and one delayed capture mode are supported.
- Recent captures are available after restart, or explicitly cut before release with a documented reason.
- Existing copy, save, OCR, annotation, R2 upload, update check, and bilingual UI behavior still work.
- Both Apple Silicon and Intel DMGs are built and smoke-tested.
